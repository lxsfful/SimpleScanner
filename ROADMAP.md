# ROADMAP · 下阶段路线图

> 下一轮迭代的候选功能。按优先级排序。
> 每个条目标了：复杂度（低/中/高）、预计耗时、依赖、风险。

---

## 🔴 P0 · 必做（1-2 周内）

### 1. 实机测试 + 算法参数调优

- **为什么**：当前所有图像处理参数是"通用默认"，没有针对真实手机拍摄场景调过
- **做什么**：
  - 拍 20+ 张不同文档（合同/收据/红头/书页/手写）
  - 记录每张的增强结果（成功/失败/部分）
  - 调 Canny 阈值、binarizer blockSize、颜色保留 HSV 阈值
  - 形成 2-3 档"算法 Profile"（清晰/普通/手写体）
- **复杂度**：低
- **耗时**：2-4 小时
- **风险**：低（只调参数）
- **依赖**：用户真机

### 2. 加 OCR 文字识别

- **为什么**：用户扫码后常要"复制文字"，OCR 是核心痛点
- **用什么**：
  - **首选：ML Kit Text Recognition v2**（Google 免费、本地、API 友好）
  - 备选：Tesseract4Android（开源但精度差）
- **做什么**：
  - 增强后跑 OCR，提取文字
  - PDF 保留可复制文本（PDFBox-Android 支持）
  - UI 加「提取文字」按钮
- **复杂度**：中
- **耗时**：1-2 天
- **风险**：
  - ML Kit 体积 +5-10 MB
  - 中文识别准确度依赖 Google 模型质量
- **依赖**：ML Kit dependency

## 🟡 P1 · 应该做（1-2 月内）

### 3. 文档管理（文件夹 / 标签 / 搜索）

- **为什么**：现在只有"按时间排"的简单历史，扫描多了找不到
- **做什么**：
  - 文件夹（自定义名称）
  - 标签（自动 + 手动：合同/收据/笔记）
  - 全文搜索（依赖 OCR 步骤 2）
- **复杂度**：中-高
- **耗时**：1-2 周
- **风险**：数据库设计（Room？SQLDelight？）

### 4. GitHub Actions CI

- **为什么**：开源项目没 CI 不专业
- **做什么**：
  - 跑 `./gradlew test assembleDebug` 每次 PR
  - 上传 APK artifact
- **复杂度**：低
- **耗时**：1-2 小时
- **依赖**：GHA 免费额度（公开仓库无限）

### 5. 应用图标重新设计

- **为什么**：现在用 vector 临时凑数
- **做什么**：
  - 找一个设计师（或者 Figma AI 工具）
  - 512×512 mipmap 全套
  - 适配 monochrome（Android 13 主题图标）
- **复杂度**：低
- **耗时**：2-3 小时（含设计）

### 6. 横屏适配

- **为什么**：现在只支持竖屏
- **做什么**：
  - Edit / Preview 屏横屏布局
  - Capture 屏横屏全屏拍照
- **复杂度**：低
- **耗时**：半天

### 7. 深色模式微调

- **为什么**：Material 3 自动适配，但有些细节可能没考虑
- **做什么**：
  - 真机测两套主题
  - 微调颜色对比度
- **复杂度**：低
- **耗时**：2-3 小时

## 🟢 P2 · 锦上添花（看需求）

### 8. 文档导出多种格式

- 当前只 PDF，可加：
  - **JPG 多张打包 ZIP**（适合发邮件）
  - **Word (.docx)**（可编辑，用 Apache POI）
  - **纯文本**（依赖 OCR）
- **复杂度**：中
- **耗时**：1-2 天

### 9. iOS 版（Compose Multiplatform）

- **为什么**：Kotlin Multiplatform + Compose Multiplatform 已经很成熟
- **做什么**：
  - 把 `image/` 包用 Kotlin/C 写
  - `ui/` 共享 Compose 代码
  - `camera/` `pdf/` `storage/` 平台特化
- **复杂度**：高
- **耗时**：2-4 周
- **风险**：iOS 端 OpenCV 不如 Android 成熟，可能要换 Core Image

### 10. 桌面版（Windows / macOS）

- **用 Compose Multiplatform Desktop** 或 **Flutter**
- **复杂度**：高
- **耗时**：1-2 月

### 11. 加密 + 密码保护

- **为什么**：扫描身份证 / 银行卡等敏感信息
- **做什么**：
  - PDF 加密（PDFBox 支持）
  - App 启动密码
  - 应用图标隐藏
- **复杂度**：中
- **耗时**：2-3 天
- **风险**：忘记密码 = 数据丢失

### 12. 云同步（私人网盘 / WebDAV / S3）

- **为什么**：多设备同步
- **做什么**：
  - 用户配 S3/WebDAV 凭证
  - 增量同步
- **复杂度**：高
- **耗时**：1-2 周
- **风险**：用户隐私 / 凭证安全

### 13. Play Store 上架

- **做什么**：
  - 生成正式 keystore
  - Play Console 账号（$25 一次性）
  - 隐私政策 / 数据安全声明
  - 上架审核（1-7 天）
- **复杂度**：中
- **耗时**：1-2 天（含等待审核）

---

## ❌ 不会做

- ❌ **账号体系**：用户希望零账号
- ❌ **社交分享**（朋友圈/小红书）：超出扫描 App 范围
- ❌ **云 OCR**：隐私考虑 + 成本
- ❌ **AR 文档增强**（自动识别表格/公式）：复杂度爆炸
- ❌ **手写识别**（区别于 OCR）：另一个量级的 ML 模型
- ❌ **多语言界面**：先做中英，需求少
